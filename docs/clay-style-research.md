# 粘土质感风格调研与本项目落地

## 调研范围与方法

- 时间：2026-02-27
- 工具：`agent-browser`
- 目标：提炼“更接近 NS 版《塞尔达传说：梦见岛》”的粘土模型观感，并映射到当前仓库（终端 + Web 渲染）

## 目标风格要点

### 1) 微缩玩具场景感

- 画面应呈现“模型台”观感：主体清晰，空间层次明显
- 棋盘是规则系统载体，视觉强化不能压过规则可读性

### 2) 柔和但有方向性的光照

- 采用顶光 + 侧光，保留卡片边缘和落地阴影
- 避免过强高光，维持哑光、柔和、偏手工材质的亮度分布

### 3) 轻量后处理与可读性保护

- 保留适度 bloom / 景深（Bokeh），避免额外移轴模糊干扰
- 文本密度上升时自动降低特效强度，优先保障规则文本可读性

## 与当前仓库映射

### 现状（已具备）

- Web 端已有透视相机、阴影、后处理（Bloom + Bokeh）
- text 与其他卡片已统一走同一渲染与后处理管线
- 已有顶光、落地阴影、移动插值、落地反馈
- 已有单一固定视觉 preset，便于统一调参与稳定输出

### 当前约束

- 不改动 `src/logic` 规则语义
- 视觉调整集中在 `src/view` / `src/web`
- 不提供运行时风格 preset 切换

## 落地建议

### P0（当前保持）

- 继续使用单一固定 `CLAY_PRESET`
- 继续使用分档镜头策略（tight / standard / wide）
- 继续启用文字密度驱动的特效衰减

### P1（可选增强）

- 卡片材质增加轻微粗糙度变化，减少“塑料片”观感
- 地面纹理引入更细颗粒噪声，增强手工模型感
- 细化色相分组，降低同场景高饱和冲突

### P2（终端等价表达）

- 终端主题继续坚持高可读性，不追求伪 3D
- 通过 ANSI 明暗层和交互高亮模拟“景深层级”

## 验收标准

- Web：规则文本与可推动对象在全局后处理下持续可辨识
- Web：`pnpm type-check` 通过，交互无明显卡顿
- 终端：规则区可读性不低于默认主题
